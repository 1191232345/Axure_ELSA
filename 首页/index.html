库存消耗预测曲线

计算逻辑
系统根据当前库存、在途库存和未来N周的累计需求预测，计算剩余库存曲线，并预测断货时间和建议补货量。
计算步骤：
  1. 获取当前库存和在途库存
  2. 计算总库存 = 当前库存 + 在途库存
  3. 获取未来N周的累计需求预测
  4. 计算每周剩余库存 = max(0, 总库存 - 累计需求)
  5. 查找剩余库存 <= 0 的第一周作为断货周
  6. 计算建议补货量 = 最后一周累计需求 - 总库存 + 最后一周安全库存
公式：
总库存 = 当前库存 + 在途库存
剩余库存[i] = max(0, 总库存 - 累计需求[i])

未来N周需求预测

计算逻辑
系统基于历史销售数据，结合季节性调整因子和促销活动影响，计算未来N周的需求预测。
计算步骤：
  1. 获取历史销售数据，计算基础需求
  2. 应用季节性调整因子到基础需求
  3. 叠加促销活动影响
  4. 计算最终预测需求 = 基础需求 × 季节性调整 + 促销影响
  5. 对每周的预测需求进行平滑处理，减少波动
  6. 输出未来N周的预测需求数据
公式：
最终预测需求[i] = 基础需求[i] × 季节性调整[i] + 促销影响[i]
其中：基础需求基于历史平均销量，季节性调整为季节性系数，促销影响为促销增量

断货风险预警

计算逻辑
系统根据每个SKU的当前库存、在途库存和未来N周的预测需求，计算断货风险百分比，并按风险等级进行排序展示。
计算步骤：
  1. 获取每个SKU的当前库存和在途库存
  2. 获取未来N周的预测需求总量
  3. 计算总可用库存 = 当前库存 + 在途库存
  4. 计算库存缺口 = 预测需求 - 总可用库存
  5. 计算风险百分比 = (库存缺口 / 预测需求) × 100%
  6. 如果风险百分比 < 0，则设为0（表示无风险）
  7. 按风险百分比从高到低排序，展示前N个高风险SKU
公式：
总可用库存 = 当前库存 + 在途库存
库存缺口 = 预测需求 - 总可用库存
风险百分比 = max(0, (库存缺口 / 预测需求) × 100%)

SKU级别补货建议报表

计算逻辑
系统为每个SKU计算补货建议，综合考虑当前库存、在途库存、预测需求和安全库存，生成详细的补货建议报表。
计算步骤：
  1. 获取每个SKU的当前库存、在途库存和未来N周预测需求
  2. 计算总可用库存 = 当前库存 + 在途库存
  3. 计算安全库存 = 预测需求 × 安全库存系数
  4. 计算建议补货量 = max(0, 预测需求 - 总可用库存 + 安全库存)
  5. 如果建议补货量小于最小补货量，则设为0（无需补货）
  6. 计算风险百分比 = (预测需求 - 总可用库存) / 预测需求 × 100%
  7. 根据风险百分比划分风险等级：高风险(>60%)、中风险(30-60%)、低风险(<30%)
  8. 根据补货提前期计算建议补货时点
公式：
总可用库存 = 当前库存 + 在途库存
安全库存 = 预测需求 × 安全库存系数
建议补货量 = max(0, 预测需求 - 总可用库存 + 安全库存)
风险百分比 = max(0, (预测需求 - 总可用库存) / 预测需求 × 100%)
