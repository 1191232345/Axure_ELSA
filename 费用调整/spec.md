# 费用调整项目规格文档

## 1. 项目概述

本项目为费用调整系统，包含未出账订单重计和出库单费用明细调整两个核心功能模块。系统使用HTML、Tailwind CSS和JavaScript实现，提供直观的用户界面和完整的交互功能。

## 2. 页面结构

### 2.1 未出账订单页面 (recalculation.html)

**主要区域：**
- 页面标题：显示"未出账订单"和"批量重计费用"描述
- 搜索筛选区域：包含订单号、客户代码、仓库、配送渠道等筛选条件
- 操作按钮区域：批量重计、导出、重计明细等功能按钮
- 订单列表区域：展示未出账订单数据，支持全选和批量操作
- 分页控制：支持分页导航和每页显示数量调整
- 逻辑说明区域：包含PRD文档和测试用例，支持展开/收起

**核心功能：**
- 订单数据筛选和搜索
- 批量重计费用
- 重计明细查询
- 数据导出

### 2.2 费用调整页面 (order-audit.html)

**主要区域：**
- 页面标题：显示"费用调整"和"出库单费用明细调整"描述
- 搜索区域：输入出库单号进行查询
- 功能按钮区域：新增费用项、批量操作等按钮
- 尺寸信息区域：显示包裹长宽高、体积重和计费重
- 费用明细表格：展示费用项列表，支持新增、编辑、删除操作
- 逻辑说明区域：包含PRD文档和测试用例，支持展开/收起

**核心功能：**
- 费用项管理（新增、编辑、删除）
- 批量操作（覆盖、更新、累加）
- 文件上传功能
- 尺寸信息计算

## 3. 技术实现

### 3.1 前端技术栈

- **HTML5**：页面结构
- **Tailwind CSS v3**：样式框架，使用自定义配置
- **JavaScript**：交互逻辑
- **Marked.js**：Markdown解析，用于PRD和测试用例展示
- **Mermaid.js**：流程图渲染，用于PRD中的用户流程图

### 3.2 核心功能实现

#### 3.2.1 未出账订单

- **数据加载**：使用本地存储模拟数据持久化
- **批量重计**：支持选择部分订单或全部订单进行重计，包含数据量超限提示
- **重计明细**：按状态筛选重计记录，支持分页查询
- **事件处理**：使用事件委托优化事件绑定，提高性能

#### 3.2.2 费用调整

- **费用项管理**：支持新增、编辑、删除费用项，包含数据验证
- **批量操作**：支持三种操作方式（覆盖、更新、累加），通过文件上传实现
- **尺寸计算**：自动计算体积重和计费重
- **文件上传**：支持Excel和CSV文件，包含文件类型和大小验证

## 5. 界面设计

### 5.1 设计规范

- **颜色方案**：
  - 主色：#3b82f6（蓝色）
  - 辅助色：#10b981（绿色）、#f59e0b（黄色）、#ef4444（红色）
  - 中性色：#f3f4f6（浅灰）、#e5e7eb（中灰）、#6b7280（深灰）

- **字体**：
  - 主字体：系统默认无衬线字体
  - 标题：1.5rem（24px），font-semibold
  - 正文：0.875rem（14px），font-normal

- **组件样式**：
  - 按钮：使用`erp-btn`系列类，包含主按钮、次要按钮、警告按钮等
  - 输入框：使用`form-input`类，统一输入控件样式
  - 卡片：使用`search-card`、`action-card`、`table-card`等类，实现卡片化布局
  - 表格：使用响应式表格，支持横向滚动

### 5.2 响应式设计

- **桌面端**（≥1024px）：完整布局，多列展示
- **平板端**（768px-1023px）：适当调整布局，保持核心功能可见
- **移动端**（<768px）：单列布局，折叠搜索区域，优化触摸交互

## 6. 交互流程

### 6.1 未出账订单流程

1. 用户进入页面，系统加载默认数据
2. 用户输入筛选条件，点击搜索按钮
3. 系统显示符合条件的订单列表
4. 用户选择订单，点击批量重计按钮
5. 系统弹出确认弹窗，用户确认后提交重计任务
6. 系统显示重计进度和结果
7. 用户可点击重计明细查看详细记录

### 6.2 费用调整流程

1. 用户进入页面，输入出库单号并搜索
2. 系统显示费用明细和尺寸信息
3. 用户可通过以下方式操作：
   - 点击"新增费用项"添加新的费用
   - 点击编辑按钮修改现有费用
   - 点击删除按钮删除费用项
   - 点击"批量操作"上传文件进行批量调整
4. 系统实时更新费用数据和尺寸信息

## 8. 性能优化

- **事件委托**：使用事件委托减少事件监听器数量
- **DOM操作优化**：使用DocumentFragment减少DOM重排重绘
- **本地存储**：使用localStorage缓存数据，减少重复计算
- **延迟加载**：PRD和测试用例内容在展开时才加载
- **图片优化**：使用适当大小的图标，减少资源加载时间

## 9. 兼容性

- **浏览器兼容**：支持Chrome、Firefox、Safari、Edge等主流浏览器
- **分辨率兼容**：适配1920x1080、1366x768等常见分辨率
- **响应式兼容**：适配桌面、平板、移动端设备
