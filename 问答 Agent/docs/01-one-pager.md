# 多轮知识问答助手 — 项目简报（One-pager）

> **版本：** v0.2 · **日期：** 2026-06-26 · **负责人：** _填写姓名_  
> **Demo 垂直场景：** 海外仓业务知识（作品集叙事用，非生产系统）

---

## 1. 一句话描述

一款 **个人作品集 Demo**：基于 RAG 的 **多轮对话问答助手**，在聊天窗中完成连续追问、引用溯源与流式回答；以海外仓 FAQ 为示例知识域，展示 AI 产品从场景到 Eval 的完整练手闭环。

---

## 2. 背景与动机

| 维度 | 说明 |
|------|------|
| **为什么做** | 转行通用 AI 产品，需要用 vibe coding 做出 **可演示、可讲清楚、可量化** 的个人项目 |
| **差异化** | 不是 ChatGPT 套壳——有 **私有知识库 RAG + 多轮上下文 + 引用 + Eval** |
| **Demo 场景** | 海外仓规则（入库/出库/计费/合规）——借自身 ToB 背景，让面试官看到领域理解 |
| **展示对象** | 面试官、同行 PM、GitHub / 个人站点访客 |

| 用户 | 场景 | 诉求 |

---

## 4. 产品目标

### Demo 交付目标（4 周）

- [ ] **多轮对话**：同一会话内至少稳定支持 **5 轮**上下文追问（如「美国仓呢？」「那计费呢？」）
- [ ] **流式回答**：Bot 回复打字机效果，体验接近真实 AI 助手
- [ ] **引用溯源**：每条事实性回答带 **1–3 条**文档引用，可展开原文
- [ ] **可演示**：3 条脚本化 Demo 路径（Happy Path / 多轮追问 / 拒答）+ 录屏
- [ ] **Eval 佐证**：≥ 30 条测试集 + 1 份指标报告（作品集附件）

### MVP 功能范围

| 做 | 不做 |
|----|------|
| 聊天对话窗（多轮 + 流式） | 登录 / 多租户 |
| 本地会话（localStorage） | WMS / 业务系统对接 |
| 预置知识库（Markdown） | 复杂文档管理后台 |
| 引用卡片 + 拒答 | 生产部署与 SLA |
| 简版 Eval 脚本 | Fine-tune |

## 5. 成功指标

| 类型 | 指标 | Demo 目标 | 说明 |
|------|------|-----------|------|
| **体验（北极星）** | 多轮追问成功率 | ≥ 80% | 10 组多轮脚本中，上下文继承正确 |
| **质量** | Eval Answer Accuracy | ≥ 80% | 个人 Demo 门槛略低于生产 |
| **质量** | Faithfulness | ≥ 85% | 引用可支撑答案 |
| **演示** | Demo 脚本通过率 | 3/3 | 录屏必备 |
| **体验** | 首 token 延迟 | ≤ 3s | 流式感知 |

## 6. 核心方案

```
┌─────────────────────────────────────────────────────────┐
│  前端 Demo（animal-island-ui + Typewriter 流式）         │
│  · 多轮会话 state · 消息列表 · 引用 Collapse · 示例问题    │
└───────────────────────────┬─────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────┐
│  RAG 服务（本地 / 轻量 API）                               │
│  Query + 最近 N 轮 history → 检索 → 生成 → citations      │
└───────────────────────────┬─────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────┐
│  预置知识库（5–8 份海外仓 Markdown / 公开资料脱敏）         │
└─────────────────────────────────────────────────────────┘
```

**多轮对话策略（产品必做）：**

1. 每轮将 **最近 5 轮** `(user, assistant)` 传入 LLM
2. 追问时 **Query 改写**（可选）：「那德国仓呢？」→「德国仓退货处理周期是多久？」
3. 检索与生成均感知 **会话实体**（国家、仓别、业务域）

---

## 7. 技术 / 设计选型

| 层 | 选择 | 理由 |
|----|------|------|
| **UI** | `animal-island-ui` | 个人 Demo 差异化视觉；Typewriter 天然适配流式 |
| **前端** | React + Vite | 与组件库一致 |
| **RAG** | 本地 Python/Node 或 Dify 自建 | 练手优先跑通 |
| **LLM** | _OpenAI / DeepSeek / Claude_ | API 即可 |
| **许可** | animal-island-ui 仅 **非商用 Demo** | 不进入企业生产环境 |

---

## 8. 明确不做

- ❌ ToB 生产系统 / 企业权限 / 审计
- ❌ 实时查库存、下单等工具调用
- ❌ 完整知识库 CMS（MVP 用预置文档 + 手动重建索引）
- ❌ 无引用的自由发挥
