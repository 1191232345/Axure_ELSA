## 2. 用户与场景

### 2.1 用户画像

**画像：Demo 访客（面试官 / 同行）**

- 打开链接，30 秒内理解「这是什么」
- 愿意试 2–3 轮追问，看 AI 是否「记得上文」
- 看到引用会加分；胡说八道会立即减分

### 2.2 用户故事

| ID | 作为… | 我想要… | 以便… | 优先级 |
|----|-------|---------|-------|--------|
| US-01 | 访客 | 在聊天窗自然语言提问 | 快速体验产品 | P0 |
| US-02 | 访客 | **在同一会话连续追问**（不换话题） | 感受多轮 AI 能力 | **P0** |
| US-03 | 访客 | 看到流式打字回答 | 体验接近真实助手 | P0 |
| US-04 | 访客 | 看到答案下的引用来源 | 相信不是瞎编 | P0 |
| US-05 | 访客 | 点击示例问题开始对话 | 降低冷启动成本 | P1 |
| US-06 | 开发者 | 新建 / 切换会话 | 演示不同场景 | P1 |
| US-07 | 开发者 | 跑 Eval 看准确率报告 | 作品集有数据支撑 | P1 |
| US-08 | 访客 | 问文档外问题时收到拒答 | 看到产品边界感 | P0 |

### 2.3 典型场景（Demo 脚本 · 必过）

#### UC-01：首轮问答（Happy Path）

```
1. 打开 Demo → 欢迎语 + 3 个示例问题
2. 点击「美国仓标准件入库截单时间是几点？」
3. Bot 流式输出答案 + 引用卡片
预期：答案正确，有引用
```

#### UC-02：多轮追问（核心 Demo）

```
1. 用户：「英国仓和德国仓退货周期有什么区别？」
2. Bot：分别说明两国周期 + 引用
3. 用户：「那德国仓的质检标准呢？」          ← 省略主语
4. Bot：仅回答德国仓质检，不混淆英国
5. 用户：「超过 7 天怎么处理？」              ← 指代德国仓退货
预期：3 轮上下文实体继承正确
```

#### UC-03：话题切换

```
1. 用户问完退货流程
2. 用户：「换个话题，超期库存怎么计费？」
3. Bot：正确切换至计费文档，不受上文退货干扰
预期：识别话题切换或仍能正确检索
```

#### UC-04：拒答

```
1. 用户：「今天美国仓总库存多少？」
2. Bot：明确拒答，不编造数字
预期：拒答文案友好（动森风提示框）
```

```

│ ChatPage（React + animal-island-ui）              │

                     │ POST /chat { messages[], sessionId }
┌────────────────────▼─────────────────────────────┐
│ Chat API                                        │
│  1. 取最近 N 轮 history                          │

                     │

└──────────────────────────────────────────────────┘
```

## 4. 功能详述

### 4.1 F1 多轮聊天窗

#### 页面结构（animal-island-ui）

```
┌─────────────────────────────────────────────────┐
│ Title：岛民知识助手 🏝          [＋ 新对话]      │
├──────────┬──────────────────────────────────────┤
│ 会话列表  │  [Bot] 你好！我是岛民知识助手…        │
│ · 对话 1 │       [示例问题 Chip × 3]             │
│ · 对话 2 │                                       │
│          │  [User] 英国仓和德国仓退货周期区别？   │
│          │  [Bot]  Typewriter 流式回答…          │
│          │        ┌ Collapse: 引用来源 ─────┐    │
│          │        │ 📄 欧洲仓退货 SOP 4.1   │    │
│          │        └─────────────────────────┘    │
│          │  [User] 那德国仓的质检标准呢？         │
│          │  [Bot]  …                            │
├──────────┴──────────────────────────────────────┤
│  Input + Button「发送」                          │
└─────────────────────────────────────────────────┘
```

#### 多轮上下文规则（产品定义）

| 规则 ID | 描述 |
|---------|------|
| MT-01 | 每请求携带 `messages[]` 全量或最近 **10 条**（5 轮） |
| MT-02 | 前端 `sessionId` 持久化 localStorage |
| MT-03 | 切换会话时恢复历史消息，不清空向量库 |
| MT-04 | 生成中禁用发送；支持 **停止生成**（P1） |
| MT-05 | Bot 消息用 `Typewriter`；**同一条消息流式 append 时不重启动画**（直接更新 text） |
| MT-06 | 用户消息立即上屏；Bot 占位 `Loading` 或 typing indicator |

#### 交互规则

| 规则 ID | 描述 |
|---------|------|
| CHAT-01 | Enter 发送，Shift+Enter 换行 |
| CHAT-02 | Markdown 渲染：列表、加粗、代码 |
| CHAT-03 | 单条引用默认展示 2 条，其余 Collapse 折叠 |

---

### 4.2 F2 流式输出

| 项 | 方案 |
|----|------|
| 传输 | SSE 或 fetch readable stream |
| 前端 | 流式阶段：逐 chunk 更新 text；完成后挂载 citations |
| 组件 | 完成后可选 `Typewriter` 回放（Demo 录屏用）；实时流式时直接 render text |
| 体验 | 首 token ≤ 3s，否则展示 Loading |

## 5. AI 方案要点

### 5.1 多轮 RAG Prompt 结构

```
[System]
你是海外仓知识助手。仅根据参考文档回答。
多轮对话时：继承用户上文提到的国家/仓别/ topic；指代不清时先简要确认。

[History]
User: 英国仓和德国仓退货周期区别？
Assistant: …
User: 那德国仓的质检标准呢？

[Retrieved Context]
chunk1…
chunk2…

[Current Query]
（可选改写后的完整问题）
```

### 5.2 Query 改写（推荐）

追问句「那德国仓呢？」→ 结合 history 改写为独立检索 query，提升检索 Hit 率。
