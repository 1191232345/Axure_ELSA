# 费用补收流程 需求澄清

**日期**: 2026-06-24
**状态**: 已确认

## 背景与目标

解决客户在仓储服务中因尺寸测量差异导致的费用补收问题，提供从真尺寸输入到补收确认/申诉的完整自助流程，提升费用透明度和客户体验。

## 范围

### In Scope（本期做）

- 真尺寸输入功能（测量建模数据录入）
- TMS 费用重计展示（基于新尺寸重新计算仓储费用）
- 补收金额计算和展示
- 客户确认补收操作（生成费用补收账单）
- 客户发起申诉操作（创建售后工单）
- 流程状态跟踪和历史记录查看

### Out of Scope（本期不做）

- 后端真实 API 对接（TMS 系统、账单系统、工单系统）
- 支付功能集成
- 管理员审核流程
- 批量处理多个订单的补收
- 退款流程（仅支持申诉）

## 用户与场景

| 角色 | 核心场景 | 频率 |
|------|----------|------|
| 客户 | 收到补收通知后，登录系统查看详情并处理 | 中 |
| 客户 | 输入或确认货物的真实尺寸信息 | 高 |
| 客户 | 查看费用重算结果和补收金额 | 高 |
| 客户 | 确认补收并生成账单，或发起申诉 | 高 |
| 客户 | 查看历史补收记录和处理状态 | 中 |

## 业务流程

```
开始 → 输入真尺寸(测量建模) → 费用重计(TMS) → 计算补收 → 确认补收？
                                                                    ↓是
                                                              新增账单(类型=费用补收) → 结束
                                                                    ↓否
                                                              发起申诉(售后工单) → 结束
```

## 核心页面/模块

1. **补收列表页** - 展示待处理的补收任务列表
2. **补收详情页** - 展示完整流程：原尺寸→新尺寸→费用对比→补收金额
3. **真尺寸输入表单** - 录入或修改货物真实尺寸（长/宽/高/重量）
4. **费用重计结果页** - 展示 TMS 重算后的费用明细
5. **确认/申诉操作区** - 提供确认补收或发起申诉的操作按钮
6. **申诉工单表单** - 填写申诉理由和上传证据

## 约束与依赖

| 类型 | 说明 |
|------|------|
| 系统依赖 | TMS 系统（模拟）、账单系统（模拟）、工单系统（模拟） |
| 数据依赖 | JSON 模拟数据文件（待创建） |
| 业务规则 | 补收金额 = 新费用 - 原已收费用；仅正向补收（不涉及退款） |
| 技术约束 | 纯静态原型，使用 localStorage 存储交互状态 |

## 待确认问题

| # | 问题 | 影响 | 建议方案 | 状态 |
|---|------|------|----------|------|
| 1 | 补收是否允许部分确认？ | P1 | 本期不支持，需全额确认或全额申诉 | 已确认-不支持 |
| 2 | 申诉后是否可以再次确认？ | P1 | 本期不支持，申诉进入工单流程后不可撤回 | 已确认-不支持 |
| 3 | 是否需要上传尺寸证明图片？ | P2 | 支持，在真尺寸输入环节可选上传 | 已确认-支持 |
| 4 | 补收是否有时效限制？ | P2 | 原型中显示倒计时，但不强制校验 | 已确认-显示即可 |

## 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 客户对补收金额争议大 | 高 | 提供详细费用对比明细；支持申诉通道 |
| 尺寸测量标准不一致 | 中 | 在界面明确标注测量规范说明 |
| 流程复杂导致客户流失 | 中 | 分步骤引导，每步清晰展示进度 |
