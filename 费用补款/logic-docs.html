逻辑说明文档

费用补收流程模块

本模块实现从尺寸输入到补收确认/申诉的完整业务流程，包含列表管理、详情展示、费用计算和操作处理四大核心功能。

1. 初始化页面数据展示逻辑
1.1 补收列表页加载
逻辑项 说明 数据来源 展示规则
任务列表加载 页面加载时获取当前客户的补收任务 data/supplement-data.json 默认显示全部状态任务，按创建时间倒序排列
状态统计 统计各状态下的任务数量 同上 在顶部统计卡片中显示：全部/待输入/待确认/已申诉
默认筛选定位 自动定位到有待处理的任务Tab 同上 如果有待输入或待确认任务，优先显示该状态
1.2 补收详情页加载
逻辑项 说明 数据来源 展示规则
任务详情加载 根据 task_id 加载完整任务信息 supplement_tasks[task_id] 展示所有字段信息，包括历史记录
步骤条状态 根据任务状态确定当前步骤 task.status pending_input → 步骤1高亮
pending_confirm → 步骤2高亮
confirmed/appealed → 步骤3高亮
倒计时显示 计算距离截止时间的剩余时间 task.deadline - now() <24小时显示红色并闪烁，<72小时显示橙色
费用对比数据 加载费用重算结果 task.fee_details[] 如果尚未重算，显示"等待费用重算中..."
2. 检索条件
2.1 筛选器配置
筛选项 类型 可选值 默认值 筛选逻辑
状态筛选 下拉选择框 全部 / 待输入 / 待确认 / 已完成 / 已申诉 全部（空值） 精确匹配 task.status
关键字搜索 文本输入框 任意文本 空字符串 模糊匹配 task.task_id 或 task.order_id

交互规则：

  • 点击搜索按钮或按 Enter 键触发筛选
  • 点击重置按钮清空所有筛选条件
  • 点击统计卡片可快速切换到对应状态的筛选结果
  • 多个筛选条件为 AND 关系
3. 模块按钮逻辑
3.1 补收列表页按钮
按钮 位置 触发动作 前置条件 后续操作
查看详情 每行操作列 SupplementApp.showDetail(taskId) 无 跳转到详情页，携带 task_id 参数
状态卡片 顶部统计区 设置筛选器值并刷新 无 快速过滤到对应状态的任务列表
搜索按钮 筛选栏右侧 SupplementApp.renderList() 无 根据当前筛选条件重新渲染表格
重置按钮 搜索按钮左侧 SupplementApp.resetFilter() 无 清空所有筛选条件并刷新列表
3.2 补收详情页按钮
按钮 位置 触发动作 前置条件 后续操作
返回列表 左上角导航栏 SupplementApp.showList() 无 隐藏详情视图，显示列表视图，刷新数据
编辑尺寸 尺寸对比区域右上角 SupplementApp.openDimensionModal() status in ['pending_input', 'pending_confirm'] 打开尺寸编辑弹窗，预填当前值
查看费用明细 费用汇总卡片下方 SupplementApp.toggleFeeDetail() 费用已重算完成 展开/收起费用明细表格
确认补收 底部操作区主按钮（蓝色） SupplementApp.openConfirmModal() 已输入尺寸 && 费用已重算 && status='pending_confirm' 弹出二次确认对话框
发起申诉 底部操作区次按钮（橙色边框） SupplementApp.openAppealModal() 无硬性前置条件 打开申诉表单弹窗
3.3 尺寸编辑弹窗按钮
按钮 位置 触发动作 前置条件 后续操作
保存并计算 弹窗底部右侧 SupplementApp.saveDimensions() 通过所有校验规则 关闭弹窗 → 触发TMS费用重算 → 更新详情页数据
取消 弹窗底部左侧 closeModal('dimensionModal') 无 关闭弹窗，不保存任何修改
上传图片 证明图片区域 handleEvidenceUpload(event) 文件类型、大小、数量校验通过 立即显示图片预览，支持删除
3.4 二次确认弹窗按钮
按钮 位置 触发动作 前置条件 后续操作
确认补收 弹窗底部右侧 SupplementApp.finalConfirm() 必须勾选协议条款复选框 生成账单号 → 显示成功Toast → 状态变为'confirmed' → 刷新详情
取消 弹窗底部左侧 closeModal('confirmModal') 无 关闭弹窗，返回详情页，无状态变更
协议勾选 弹窗内复选框 启用/禁用确认按钮 无 未勾选时确认按钮置灰不可点击
3.5 申诉表单弹窗按钮
按钮 位置 触发动作 前置条件 后续操作
提交申诉 弹窗底部右侧 SupplementApp.submitAppeal() 所有必填项已填写且通过校验 创建工单号 → 显示成功提示 → 状态变为'appealed' → 刷新详情
保存草稿 弹窗底部左侧 SupplementApp.saveAppealDraft() 至少填写了申诉理由 提示"草稿已保存"，关闭弹窗（原型阶段仅提示）
取消/关闭 弹窗顶部或底部 closeModal('appealModal') 有未保存内容时需确认 关闭弹窗（原型阶段直接关闭）
4. 字段取值逻辑
4.1 补收列表页字段
字段 数据来源 取值规则 显示格式
任务ID task.task_id 直接取值 SUP-YYYYMMDD-NNN 格式，蓝色加粗
订单号 task.order_id 直接取值 ORD-YYYYMMDD-NNN 格式
补收金额 task.supplement_amount 计算：recalculated_fee - original_fee ¥ XXX.XX 格式，正值红色加粗
状态 task.status 映射显示名称 彩色标签：待输入(橙)/待确认(蓝)/已完成(绿)/已申诉(红)
截止时间 task.deadline 时间戳转日期 YYYY-MM-DD，<3天橙色，过期红色
创建时间 task.created_at 时间戳转日期 YYYY-MM-DD
4.2 补收详情页字段
字段 数据来源 取值规则 显示格式
原/新长度 original_dimensions.length / actual_dimensions.length 直接取值 XX.X cm，新值蓝色加粗
尺寸差异 计算得出 (新体积-原体积) / 原体积 * 100 +XX.X% 增大红色，减小绿色
原费用 task.original_fee 直接取值 ¥ XXX.XX
新费用 task.recalculated_fee TMS 返回或前端模拟计算 ¥ XXX.XX
补收金额 task.supplement_amount new_fee - original_fee ¥ XXX.XX 大号红色加粗
剩余时间 deadline - now() 动态计算，每分钟更新 X天X小时X分钟，<24小时红色闪烁
4.3 费用明细字段
字段 数据来源 取值规则 显示格式
费用类别 fee_detail.category 枚举映射 标签样式：仓储费/操作费/其他
项目名称 fee_detail.item_name 直接取值 如"月度仓储费"、"入库操作费"
原费用小计 fee_detail.original_subtotal unit_price * quantity_original ¥ XXX.XX
新费用小计 fee_detail.new_subtotal unit_price * quantity_actual ¥ XXX.XX
差额 fee_detail.difference new_subtotal - original_subtotal +¥ XX.XX 正值红色，负值绿色
