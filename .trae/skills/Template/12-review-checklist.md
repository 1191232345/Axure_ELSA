# 评审检查清单

本文件包含需求设计、原型设计和测试用例编写的评审检查清单，确保交付质量。

## 1. PRD 评审检查清单

### 1.1 文档基本信息

- [ ] 文档标题清晰明确
- [ ] 版本号格式正确（V1.0）
- [ ] 日期格式正确（YYYY-MM-DD）
- [ ] 状态标注准确（待评审/评审中/已通过/已变更）
- [ ] 作者信息完整

### 1.2 Executive Summary 执行摘要

- [ ] 问题陈述清晰，业务背景明确
- [ ] 痛点描述具体，有数据支撑
- [ ] 解决方案可行，与问题对应
- [ ] 成功指标可量化，有明确目标值

### 1.3 User Experience 用户体验

- [ ] 用户画像完整，覆盖所有角色
- [ ] 用户旅程图清晰，步骤完整
- [ ] 用户流程图正确，分支完整
- [ ] 流程说明详细，易于理解

### 1.4 Functional Modules 功能模块

- [ ] 功能清单完整，无遗漏
- [ ] 功能描述清晰，无歧义
- [ ] 优先级设置合理（P0/P1）
- [ ] 功能模块划分合理

### 1.5 Functional Logic 功能逻辑

- [ ] 初始化逻辑描述完整
- [ ] 按钮逻辑描述清晰
- [ ] 字段取值规则明确
- [ ] 弹窗属性描述完整
- [ ] 异常处理逻辑完整

### 1.6 文档格式

- [ ] 标题层级正确（H1-H4）
- [ ] 表格格式规范
- [ ] Mermaid流程图语法正确
- [ ] 无错别字和语法错误
- [ ] 排版整洁美观

## 2. 原型评审检查清单

### 2.1 页面结构

- [ ] 页面布局合理

### 2.2 视觉设计

- [ ] 字体大小符合规范
- [ ] 间距符合设计规范
- [ ] 图标使用正确

### 2.3 交互设计

- [ ] 按钮状态完整（默认/悬停/按下/禁用）
- [ ] 输入框状态完整（默认/聚焦/错误/禁用）
- [ ] 模态框动画流畅
- [ ] 提示信息友好

### 2.4 功能完整性

- [ ] 筛选功能完整
- [ ] 按钮功能完整
- [ ] 表格展示完整
- [ ] 分页功能正常

### 2.5 逻辑说明区域

- [ ] 初始化页面逻辑描述完整
- [ ] 检索条件描述清晰
- [ ] 按钮逻辑描述完整
- [ ] 属性取值逻辑描述完整

### 2.6 逻辑说明完整性检查（重要）

**⚠️ 强制检查项 - 每个页面必须包含以下逻辑说明：**

#### 必备逻辑说明模块

- [ ] **初始化页面（数据展示逻辑）** - 必须包含表格，不能为空或"/"
- [ ] **检索条件** - 必须包含表格，不能为空或"/"
- [ ] **按钮逻辑** - 必须包含表格，不能为空或"/"
- [ ] **属性取值逻辑** - 必须包含表格，详细说明字段取值规则
- [ ] **弹窗属性描述** - 如有弹窗，必须包含弹窗字段说明

#### 常见缺失问题检查

- [ ] 检查是否有页面完全缺失逻辑说明区域
- [ ] 检查逻辑说明表格是否为空或只有"/"占位符
- [ ] 检查字段取值规则是否详细（不能只写"直接取值"）
- [ ] 检查按钮逻辑是否包含前置条件和后续操作
- [ ] 检查弹窗字段是否包含输入方式和取值规则

#### 逻辑说明质量检查

- [ ] 数据来源是否明确（系统生成/键盘输入/手工选择等）
- [ ] 取值规则是否具体（格式、默认值、验证规则等）
- [ ] 展示规则是否清晰（排序、筛选、格式化等）
- [ ] 异常处理是否说明（错误提示、边界条件等）

### 2.7 代码质量

- [ ] HTML结构语义化
- [ ] CSS样式规范
- [ ] JavaScript交互正确
- [ ] 无控制台错误

## 5. 评审流程

### 5.1 自评阶段

1. 按照检查清单逐项检查
2. 标记已完成项和待修改项
3. 修改待完善的内容
4. 确认所有检查项通过

### 5.2 评审会议

1. 准备评审材料
2. 按模块逐一讲解
3. 记录评审意见
4. 确认修改项和责任人

### 5.3 修改确认

1. 按评审意见修改
2. 更新文档版本
3. 发送修改确认通知
4. 归档评审记录

## 6. 常见问题检查

### 6.1 PRD常见问题

| 问题类型 | 检查项 |
|----------|--------|
| 逻辑漏洞 | 是否有未考虑的场景 |
| 需求模糊 | 描述是否足够具体 |
| 优先级不当 | P0功能是否为核心功能 |
| 遗漏功能 | 是否有遗漏的功能点 |

### 6.2 原型常见问题

| 问题类型 | 检查项 |
|----------|--------|
| 样式不一致 | 是否符合设计规范 |
| 交互缺失 | 是否缺少必要交互 |
| 布局问题 | 是否有布局错乱 |
| 响应式问题 | 是否适配不同屏幕 |

### 6.3 测试用例常见问题

| 问题类型 | 检查项 |
|----------|--------|
| 覆盖不足 | 是否遗漏测试场景 |
| 步骤不清 | 测试步骤是否可执行 |
| 结果模糊 | 预期结果是否明确 |
| 数据缺失 | 测试数据是否准备 |
