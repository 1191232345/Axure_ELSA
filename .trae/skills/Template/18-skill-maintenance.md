# Skills 自我维护指南（v1.0）

本文件定义 Skills 体系的自我检测、总结与更新机制，确保 skills 始终与实际代码保持同步。

---

## 触发条件

| 触发方式 | 关键词 | 说明 |
|----------|--------|------|
| 用户主动触发 | "检查skills"、"更新skills"、"skills健康检查" | 执行完整检测流程 |
| 用户主动触发 | "扫描冗余"、"检测重复" | 仅执行冗余检测 |

| 开发后可选触发 | 完成一个模块开发后 | 执行漂移检测 |

## 执行流程

### Step 1：漂移检测（Drift Detection）

检测 skills 文档中的代码定义与实际公共资源文件的差异。

#### 1.1 JS 漂移检测

```
对比对象：
  权威文档：17-common-js.md 中的代码块
  实际代码：/common/js/common.js

检测步骤：
1. 提取 17-common-js.md 中所有 function xxx() 声明
2. 提取 common.js 中所有 function xxx() 声明
3. 对比函数签名列表，输出三类差异：
   - 🆕 common.js 有但 17-common-js.md 未记录 → 需补充文档
   - 🗑️ 17-common-js.md 有但 common.js 已移除 → 需清理文档
   - ⚠️ 函数签名一致但实现差异 > 20% → 需同步更新
```

#### 1.2 CSS 漂移检测

```
对比对象：
  权威文档：16-common-css.md 中的 CSS 代码块
  实际代码：/common/css/common.css

检测步骤：
1. 提取 16-common-css.md 中所有 CSS 选择器
2. 提取 common.css 中所有 CSS 选择器
3. 对比选择器列表，输出三类差异：
   - 🆕 common.css 有但 16-common-css.md 未记录 → 需补充文档
   - 🗑️ 16-common-css.md 有但 common.css 已移除 → 需清理文档
   - ⚠️ 选择器相同但属性值差异 > 20% → 需同步更新
```

#### 1.3 设计令牌漂移检测

```
对比对象：
  权威文档：00-design-tokens.md 中的 :root 变量定义
  依赖文件：02-css-styles.md、16-common-css.md 中引用的变量

检测步骤：
1. 提取 00-design-tokens.md 中所有 --xxx: yyy 变量定义
2. 扫描 02-css-styles.md 和 16-common-css.md 中的 :root 块
3. 检测是否有文件定义了 00-design-tokens.md 中不存在的变量
4. 输出：
   - ⚠️ 补充变量（16-common-css.md 中合理保留的）→ 确认是否需要提升为全局令牌
   - ❌ 重复变量（与 00-design-tokens.md 定义冲突的）→ 必须删除
```

---

### Step 2：冗余检测（Redundancy Detection）

检测 skills 文件之间的代码重复。

#### 2.1 代码块提取与比对

```
检测步骤：
1. 扫描 Template/ 目录下所有 .md 文件
2. 提取每个文件中 ```javascript 和 ```css 代码块
3. 对代码块做文本标准化（去除注释、空白、变量名差异）
4. 计算代码块之间的相似度（基于行级比对）
5. 相似度 > 80% → 标记为疑似重复
```

#### 2.2 输出格式

```
冗余报告示例：

🔴 严重重复（相似度 > 95%）：
  - [04-javascript.md §9] showToast() 与 [17-common-js.md §4] 
    → 建议：04 引用 17，删除重复实现

🟡 中度重复（相似度 80-95%）：
  - [02-css-styles.md §3] .btn-primary 与 [06-components.md §2]
    → 建议：02 保留基础样式，06 引用 02

🟢 轻微重复（相似度 < 80%）：
  - 无
```

---

### Step 3：索引同步（Index Sync）

根据实际文件状态更新 SKILL.md。

#### 3.1 文件清单同步

```
同步步骤：
1. 扫描 Template/ 目录下所有 .md 文件
2. 读取每个文件的：
   - 一级标题（# xxx）
   - 版本号（v2.x）
   - 定位描述（唯一数据源 / 架构指南 / 精简版 等）
3. 与 SKILL.md 当前文件清单对比
4. 输出差异：
   - 🆕 新增文件 → 添加到清单
   - 🗑️ 已归档文件 → 标记删除线
   - 📝 描述变更 → 更新说明文字
```

#### 3.2 版本记录追加

```
当 Step 1-3 产生实际修改时：
1. 在 SKILL.md 版本更新记录表头追加新行
2. 格式：| v2.x | 日期 | 变更摘要 |
3. 变更摘要从本次检测结果中自动生成
```

---

### Step 4：变更记录（Changelog）

每次维护操作后记录变更。

#### 4.1 变更日志格式

```markdown
## 变更日志

### 2026-05-22 | 健康检查
**检测范围**：全量扫描
**检测结果**：
- ✅ JS 漂移：17-common-js.md 与 common.js 一致
- ⚠️ CSS 漂移：16-common-css.md 缺少 .tooltip 样式（common.css 第 245 行新增）
- ✅ 令牌漂移：无冲突
- ⚠️ 冗余检测：04-javascript.md §9 Toast 与 17-common-js.md §4 重复（相似度 92%）
- ✅ 索引同步：SKILL.md 文件清单与实际一致

**执行操作**：
1. 补充 16-common-css.md 中 .tooltip 样式文档
2. 04-javascript.md §9 Toast 改为引用 17-common-js.md §4
3. 更新 SKILL.md 版本记录至 v2.5
```

## 单一数据源权威映射

维护时必须遵循以下权威关系，任何修改不得破坏：

```
CSS 变量：
  00-design-tokens.md  ←── 唯一权威（:root 变量定义）
       ↑ 引用
  16-common-css.md     ←── 补充变量 + .prose 权威定义
       ↑ 引用
  02-css-styles.md     ←── 模块样式（变量/prose 已去重）

JS 函数：
  17-common-js.md      ←── 唯一权威（公共函数实现）
       ↑ 引用
  04-javascript.md     ←── 模块 JS（Part1 已去重，Part2-6 独有）

数据持久化：
  17-common-js.md      ←── 代码实现（APIDataManager/StateManager）
  15-data-persistence.md ←── 架构指南（配置/集成/部署）

HTML 模板：
  03-html-structure.md ←── 主页面模板
  11-testcase-html-template.md ←── 测试用例页面模板（精简版）
```
