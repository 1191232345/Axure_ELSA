# 测试用例文档规范

本文件定义TOB产品的测试用例文档规范，确保所有模块的测试用例文档保持一致的结构和质量。

> **说明**：本文档中的示例内容仅供参考，实际使用时请替换为具体业务内容。

## 1. 文档基本信息

每个测试用例文档开头应包含以下基本信息：

```markdown
# [模块名称]测试用例

**版本**: V1.0  
**日期**: YYYY-MM-DD  
**状态**: 待评审

---
```

## 2. 文档结构

### 2.1 完整目录结构

```
1. 测试概述
   1.1 测试范围
   1.2 测试类型
   1.3 测试环境

2. [模块名称]测试用例
   2.1 [子模块名称]
   2.2 [子模块名称]
   ...

N. 集成测试用例
   N.1 [流程名称]
   N.2 数据一致性
```

## 3. 测试概述

### 3.1 测试范围

描述测试覆盖的功能模块。

```markdown
### 1.1 测试范围

本文档涵盖[系统名称]的所有功能模块测试，包括：
- [模块1]
- [模块2]
- [模块3]
```

**示例**：
```markdown
### 1.1 测试范围

本文档涵盖虚拟仓管理系统的所有功能模块测试，包括：
- 组织架构管理
- 虚拟仓管理
- 库存管理
- 调拨管理
- 出库管理
- 账单管理
```

### 3.2 测试类型

定义测试类型和覆盖范围。

```markdown
### 1.2 测试类型

| 测试类型 | 说明 | 覆盖范围 |
|----------|------|----------|
| 功能测试 | 验证各功能模块是否按需求正常工作 | 所有P0/P1功能 |
| 集成测试 | 验证模块间数据流转和业务流程 | 跨模块业务流程 |

| 数据隔离测试 | 验证不同层级账号的数据访问权限 | 权限控制逻辑 |
| 用户体验测试 | 验证界面交互和操作流程 | 用户操作体验 |
```

## 4. 测试用例编号规范

### 4.1 编号格式

测试用例编号格式：`TC-[模块缩写]-[序号]`

| 模块 | 缩写 | 示例 |
|------|------|------|
| 组织架构管理 | ORG | TC-ORG-101 |
| 用户管理 | USR | TC-USR-001 |
| 订单管理 | ORD | TC-ORD-001 |
| 商品管理 | PRD | TC-PRD-001 |
| 库存管理 | INV | TC-INV-001 |
| 财务管理 | FIN | TC-FIN-001 |
| 集成测试 | INT | TC-INT-001 |

### 4.2 序号规则

- 001-099：基础操作测试（创建、编辑、删除）
- 101-199：查询筛选测试
- 201-299：权限验证测试
- 301-399：业务逻辑测试

### 4.3 优先级定义

| 优先级 | 说明 | 覆盖范围 |
|--------|------|----------|
| P0 | 核心功能，必须通过 | 核心业务流程、关键功能 |
| P1 | 重要功能，应该通过 | 重要功能、边界条件 |
| P2 | 一般功能，建议通过 | 次要功能、异常场景 |

## 7. 测试步骤编写规范

### 7.1 步骤格式

使用有序列表格式，每个步骤用 `<br>` 分隔：

```markdown
1.点击"新建"按钮<br>2.填写表单字段<br>3.点击保存
```

### 7.2 步骤动词

| 动词 | 用途 | 示例 |
|------|------|------|
| 点击 | 点击按钮、链接 | 点击"新建"按钮 |
| 填写 | 输入文本 | 填写名称字段 |
| 选择 | 下拉选择、单选 | 选择部门 |
| 勾选 | 复选框操作 | 勾选"同步迁移"选项 |
| 验证 | 检查结果 | 验证列表显示 |
| 确认 | 确认操作 | 确认删除 |

### 7.3 验证语句

使用"验证"开头的语句：

- 验证列表显示
- 验证提示信息
- 验证数据更新
- 验证状态变更
- 验证权限控制

## 8. 预期结果编写规范

### 8.1 成功场景

描述操作成功后的结果：

```markdown
创建成功，列表显示新数据
更新成功，列表显示更新后的信息
删除成功，列表不再显示
```

### 8.2 失败场景

描述操作失败后的提示：

```markdown
提示"[字段]已存在"，创建失败
提示"[字段]不能为空"，保存失败
提示"权限不足"，操作被拒绝
```

### 8.3 数据验证

描述数据状态变化：

```markdown
库存扣减，可用数量减少
状态变更为"已完成"
数据流转正确，状态准确
```
