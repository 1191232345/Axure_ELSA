# 检查清单

评审检查清单 + 安全检查清单，确保交付质量和部署安全。

---

## Part 1：评审检查清单

### 1. PRD 评审

**文档基本信息**
- [ ] 文档标题清晰明确
- [ ] 版本号格式正确（V1.0）
- [ ] 日期格式正确（YYYY-MM-DD）
- [ ] 状态标注准确
- [ ] 作者信息完整

**Executive Summary**
- [ ] 问题陈述清晰，业务背景明确
- [ ] 痛点描述具体，有数据支撑
- [ ] 解决方案可行，与问题对应
- [ ] 成功指标可量化，有明确目标值

**功能模块**
- [ ] 功能清单完整，无遗漏
- [ ] 功能描述清晰，无歧义
- [ ] 优先级设置合理（P0/P1）
- [ ] 功能模块划分合理

**功能逻辑**
- [ ] 初始化逻辑描述完整
- [ ] 按钮逻辑描述清晰
- [ ] 字段取值规则明确
- [ ] 弹窗属性描述完整
- [ ] 异常处理逻辑完整

**文档格式**
- [ ] 标题层级正确（H1-H4）
- [ ] 表格格式规范
- [ ] Mermaid流程图语法正确
- [ ] 无错别字和语法错误

### 2. 原型评审

**页面结构**
- [ ] 页面标题与PRD一致
- [ ] 导航结构正确
- [ ] 页面布局合理
- [ ] 响应式设计适配

**视觉设计**
- [ ] 主色调使用正确（#2a3b7d）
- [ ] 字体大小符合规范
- [ ] 间距符合设计规范
- [ ] 图标使用正确（Font Awesome 6）

**交互设计**
- [ ] 按钮状态完整（默认/悬停/按下/禁用）
- [ ] 输入框状态完整（默认/聚焦/错误/禁用）
- [ ] 模态框动画流畅
- [ ] 提示信息友好

**功能完整性**
- [ ] 筛选功能完整
- [ ] 按钮功能完整
- [ ] 表格展示完整
- [ ] 分页功能正常

**逻辑说明（强制检查项）**
- [ ] 初始化页面逻辑描述完整（不能为空或"/"）
- [ ] 检索条件描述清晰
- [ ] 按钮逻辑包含前置条件和后续操作
- [ ] 属性取值逻辑详细（不能只写"直接取值"）
- [ ] 弹窗字段包含输入方式和取值规则

**代码质量**
- [ ] HTML结构语义化
- [ ] CSS样式规范
- [ ] JavaScript交互正确
- [ ] 无控制台错误

### 3. 测试用例评审

- [ ] 用例编号规范统一
- [ ] 测试项描述清晰
- [ ] 前置条件完整
- [ ] 测试步骤可执行
- [ ] 预期结果明确
- [ ] 优先级设置合理
- [ ] 正向/异常/边界场景覆盖完整

### 4. 一致性检查

- [ ] PRD与原型：功能点一一对应，字段/按钮名称一致
- [ ] PRD与测试用例：功能模块对应，业务规则一致

## Part 2：安全检查清单

> **重要提醒**：GitHub Pages 和 Vercel 默认部署到公开 URL！部署前必须完成以下检查。

### 1. 数据安全

- [ ] 无真实用户数据（PII：姓名/邮箱/手机号）
- [ ] 无生产数据库内容
- [ ] 使用虚构测试数据（如 `张三（示例）`、`demo@example.com`）

### 2. 凭证安全

- [ ] 无 API keys / tokens / secrets 硬编码
- [ ] 无用户名密码硬编码
- [ ] 无数据库连接串
- [ ] 无 SSH keys（.pem/.key 文件）

**禁止**：
```javascript
const API_KEY = "sk-xxxxx";  // ❌
const INTERNAL_URL = "http://192.168.1.100:8080";  // ❌
```

**推荐**：
```javascript
const API_BASE = "/api/data";  // ✅ 相对路径
const DATA_FILE = "data/test-data.json";  // ✅ 本地文件
```

### 3. 内容安全

- [ ] 无内部 URL（192.168.x.x / 10.x.x.x）
- [ ] 无内部工具截图
- [ ] 无未发布功能细节
- [ ] 无商业机密信息（定价策略/客户名单/核心算法）

### 4. 依赖安全

- [ ] 不引用未知 CDN（仅使用 jsdelivr/cloudflare/cdnjs）
- [ ] 无跟踪脚本（analytics/tracking）
- [ ] 无广告脚本
- [ ] 无恶意脚本（eval/Function 构造）

### 5. 自动化检查脚本

```bash
#!/bin/bash
echo "🔍 开始安全检查..."
grep -r "api_key\|apikey\|secret\|password\|token" --include="*.js" --include="*.html" .
grep -r "192\.168\|10\.0\|172\.16" --include="*.html" --include="*.js" .
grep -r "script src=" --include="*.html" . | grep -v "cdn.jsdelivr\|cdn.tailwind\|cdnjs"
echo "✅ 安全检查完成"
```

### 6. 违规处理

发现违规项时：立即停止部署 → 标记违规内容 → 替换为安全内容 → 重新检查 → 确认通过后部署
