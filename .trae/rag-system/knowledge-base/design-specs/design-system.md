# 设计系统规范

## 1. 设计理念

### 核心原则
- **一致性**: 确保所有界面元素在视觉和交互上保持一致
- **清晰性**: 界面布局清晰，信息层次分明
- **易用性**: 操作流程简单直观，减少用户认知负担
- **响应式**: 适配不同设备尺寸，提供良好的跨设备体验

### 设计风格
- **主色调**: #2a3b7d（深蓝色）
- **辅助色**: #36CFC9（青色）、#722ED1（紫色）
- **功能色**: #00B42A（成功）、#FF7D00（警告）、#F53F3F（危险）
- **中性色**: #1D2129（深色）、#374151（文本）、#6B7280（次要文本）、#E5E7EB（边框）、#F9FAFB（背景）

## 2. 排版规范

### 字体
- **主要字体**: Inter, system-ui, sans-serif
- **代码字体**: Monaco, Menlo, monospace

### 字号规范
- **页面标题**: 24px, 700 权重
- **卡片标题**: 18px, 600 权重
- **section 标题**: 16px, 600 权重
- **正文文本**: 14px, 400 权重
- **辅助文本**: 12px, 400 权重

### 行高规范
- **标题行高**: 1.2
- **正文字高**: 1.5
- **辅助文本行高**: 1.4

## 3. 布局规范

### 容器规范
- **页面容器**: 最大宽度 1280px，居中对齐
- **响应式断点**:
  - 移动端: < 640px
  - 平板: 640px - 1024px
  - 桌面: > 1024px

### 间距规范
- **宏观间距**: 16px（页面边距）
- **中观间距**: 12px（区域间距）
- **微观间距**: 8px（元素间距）
- **紧凑间距**: 4px（文本与图标间距）

### 圆角规范
- **大圆角**: 16px（模态框、卡片）
- **中圆角**: 12px（按钮、输入框）
- **小圆角**: 8px（标签、小部件）
- **最小圆角**: 4px（徽章）

### 整体布局结构
- **顶部导航栏**: 固定在页面顶部，包含系统名称和主导航菜单
- **标签切换系统**: 固定在右下角，用于切换原型、PRD、测试用例三个模块
- **主内容区域**: 根据所选标签显示不同内容
- **响应式设计**: 适配不同屏幕尺寸

### 模块布局
- **原型模块**: 显示实际的业务功能界面，采用卡片式布局
- **PRD模块**: 从Markdown文件加载并渲染PRD文档，包含左侧目录导航
- **测试用例模块**: 从Markdown文件加载并渲染测试用例，包含左侧目录导航

## 4. 组件规范

### 按钮
- **尺寸**: 大（40px）、中（32px）、小（24px）
- **样式**:
  - **主按钮**: 深蓝色背景，白色文字，hover效果
  - **次要按钮**: 白色背景，深色文字，边框效果
  - **警告按钮**: 橙色背景，白色文字
  - **危险按钮**: 红色背景，白色文字
- **状态**: 正常、悬停、点击、禁用
- **ERP风格按钮**: 特定业务场景下的按钮样式，包含图标和文字

### 输入框
- **高度**: 32px
- **样式**: 文本输入、下拉选择、日期选择
- **状态**: 正常、聚焦、错误、成功、禁用
- **搜索输入框**: 带有搜索图标的输入框

### 表格
- **行高**: 48px
- **样式**:
  - **表头**: 深蓝色渐变背景，白色文字
  - **数据行**: 交替行背景，hover效果
  - **操作列**: 固定在右侧，包含编辑、删除等按钮
- **操作**: 内联操作按钮、批量操作
- **层级展示**: 支持树形结构的展开/折叠

### 卡片
- **内边距**: 16px
- **阴影**: 轻微阴影效果
- **边框**: 1px 浅色边框
- **圆角**: 12px

### 模态框
- **宽度**: 最大宽度 600px
- **高度**: 最大高度 80vh
- **动画**: 淡入淡出效果，缩放动画
- **布局**: 包含头部、主体和底部操作区

### 导航菜单
- **顶部导航**: 深色背景，白色文字
- **下拉菜单**: 白色背景，阴影效果，hover状态
- **层级导航**: 支持多级菜单

### 标签切换
- **样式**: 固定在右下角，渐变背景
- **交互**: 点击切换，平滑过渡效果
- **状态**: 激活状态与非激活状态

## 5. 图标规范

### 图标库
- **主要图标库**: Font Awesome 4.7.0
- **图标风格**: 线性图标
- **图标尺寸**: 14px、16px、20px、24px

### 使用规范
- 图标应与文本保持一致的视觉权重
- 操作按钮中图标应位于文本左侧
- 状态指示图标应清晰易懂

## 6. 交互规范

### 动画效果
- **过渡动画**: 0.3s ease 过渡
- **悬停效果**: 轻微放大、颜色变化
- **点击效果**: 轻微下沉、阴影变化
- **加载效果**: 旋转动画、骨架屏
- **模态框动画**: 缩放动画，从中心展开
- **标签切换动画**: 平滑过渡效果
- **表格行展开/折叠动画**: 平滑高度变化

### 反馈机制
- **成功反馈**: 绿色提示、成功图标
- **错误反馈**: 红色提示、错误图标
- **警告反馈**: 黄色提示、警告图标
- **信息反馈**: 蓝色提示、信息图标
- **加载状态**: 显示加载动画和提示文本
- **操作反馈**: 按钮点击反馈，状态变化

### 键盘导航
- 所有可交互元素应支持键盘导航
- 表单应支持 Tab 键切换
- 应支持 Enter 键提交表单

### 用户体验
- **加载状态**: 显示加载动画，提供手动加载选项
- **错误处理**: 友好的错误提示，提供重试选项
- **操作反馈**: 按钮点击反馈，状态变化
- **响应式设计**: 适配不同设备，优化触摸目标

## 7. 技术实现

### 技术栈
- **CSS框架**: Tailwind CSS
- **图标库**: Font Awesome 4.7.0
- **Markdown解析**: Marked.js
- **图表渲染**: Mermaid.js

### 核心功能
- **标签切换**: 实现不同模块之间的切换
- **Markdown渲染**: 将PRD和测试用例从Markdown转换为HTML
- **Mermaid图表**: 支持流程图的渲染和放大预览
- **模态框**: 实现新增、编辑等操作的弹窗
- **表格交互**: 支持表格行的展开/折叠和操作

### 代码结构
- **HTML结构**: 语义化标签，模块化组织
- **CSS组织**: Tailwind配置，自定义工具类
- **JavaScript功能**: 标签切换，数据加载，图表渲染，模态框，表格交互

- 支持撤销操作
- 优化加载状态

### 开发实践
- 使用设计系统组件
- 遵循响应式设计原则
- 优化性能和加载速度
- 确保跨浏览器兼容性
