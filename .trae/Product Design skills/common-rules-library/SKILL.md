# 公共规则库

为产品需求文档提供标准化的规则模板，避免重复描述，提高文档质量和开发效率。

## 一、数据校验规则库

### 1.1 字段级校验规则

| 校验类型 | 规则说明 | 错误提示模板 | 适用场景 |
|---------|---------|-------------|---------|
| **必填校验** | 字段不能为空或null | "请输入/选择{字段名}" | 所有必填字段 |
| **格式校验-数字** | 必须为数字类型 | "请输入有效的数字" | 价格、数量等 |
| **格式校验-整数** | 必须为整数 | "请输入整数" | 序号、数量等 |
| **格式校验-日期** | 必须符合日期格式YYYY-MM-DD | "请选择有效的日期" | 日期字段 |
| **格式校验-时间** | 必须符合时间格式YYYY-MM-DD HH:mm:ss | "请选择有效的时间" | 时间字段 |
| **格式校验-邮箱** | 必须符合邮箱格式 | "请输入有效的邮箱地址" | 邮箱字段 |
| **格式校验-手机** | 必须符合手机号格式 | "请输入有效的手机号码" | 手机字段 |
| **范围校验-非负** | 必须≥0 | "{字段名}不能小于0" | 价格、数量等 |
| **范围校验-正数** | 必须>0 | "{字段名}必须大于0" | 数量等 |
| **范围校验-区间** | 必须在[min, max]范围内 | "{字段名}必须在{min}-{max}之间" | 有明确范围的字段 |
| **长度校验-最大** | 字符长度≤maxLength | "{字段名}不能超过{maxLength}个字符" | 文本字段 |
| **长度校验-最小** | 字符长度≥minLength | "{字段名}不能少于{minLength}个字符" | 文本字段 |
| **长度校验-固定** | 字符长度=fixedLength | "{字段名}必须为{fixedLength}个字符" | 编码、ID等 |
| **精度校验** | 小数位数≤precision | "{字段名}最多保留{precision}位小数" | 价格、金额等 |
| **唯一性校验** | 值不能重复 | "{字段名}已存在，请使用其他值" | 编码、名称等 |
| **正则校验** | 符合指定正则表达式 | "请输入正确格式的{字段名}" | 特定格式字段 |

### 1.2 表单级校验规则

| 校验类型 | 规则说明 | 错误提示 | 适用场景 |
|---------|---------|---------|---------|
| **字段关联校验** | 字段A的值必须与字段B存在关联关系 | "{字段A}与{字段B}不匹配" | 父子级字段 |
| **范围连续校验** | 数值范围必须连续，不能有间隙 | "数值范围不连续，请检查" | 阶梯价格等 |
| **范围不重叠校验** | 数值范围不能重叠 | "数值范围重叠，请检查" | 阶梯价格等 |
| **开始结束校验** | 开始值必须<结束值 | "开始值必须小于结束值" | 时间范围、数值范围 |
| **时间先后校验** | 开始时间必须≤结束时间 | "开始时间不能晚于结束时间" | 时间范围字段 |
| **逻辑依赖校验** | 字段A填写时，字段B必须填写 | "填写{字段A}时，必须填写{字段B}" | 依赖关系字段 |
| **互斥校验** | 字段A和字段B不能同时填写 | "{字段A}和{字段B}不能同时填写" | 互斥选项 |

### 1.3 业务级校验规则

| 校验类型 | 规则说明 | 错误提示 | 适用场景 |
|---------|---------|---------|---------|
| **引用完整性校验** | 删除前检查是否被其他数据引用 | "{数据}已被使用，无法删除" | 删除操作 |
| **状态约束校验** | 特定状态下不允许某些操作 | "{状态}状态下无法执行该操作" | 状态流转 |
| **权限校验** | 用户必须有相应权限才能操作 | "您没有权限执行该操作" | 所有操作 |
| **业务规则校验** | 符合特定业务规则 | "不符合业务规则：{具体规则}" | 业务逻辑 |
| **数据完整性校验** | 相关数据必须完整才能操作 | "相关数据不完整，请先完善" | 复合数据 |

## 二、异常处理规则库

### 2.1 网络异常处理

| 异常类型 | 触发条件 | 处理方式 | 提示文案 | 后续操作 |
|---------|---------|---------|---------|---------|
| **请求超时** | 请求超过timeout时间 | 显示错误提示，提供重试按钮 | "请求超时，请稍后重试" | 用户点击重试 |
| **网络断开** | 网络连接中断 | 显示离线提示，禁用操作按钮 | "网络连接已断开，请检查网络" | 恢复网络后自动刷新 |
| **服务器错误** | HTTP 500等服务器错误 | 显示错误提示，记录日志 | "服务器异常，请联系管理员" | 提供反馈入口 |
| **接口不存在** | HTTP 404 | 显示错误提示 | "请求的资源不存在" | 返回上一页 |
| **权限不足** | HTTP 403 | 显示权限提示 | "您没有权限访问该资源" | 返回首页或申请权限 |

### 2.2 数据异常处理

| 异常类型 | 触发条件 | 处理方式 | 提示文案 | 后续操作 |
|---------|---------|---------|---------|---------|
| **数据不存在** | 查询的数据已被删除 | 显示空状态或错误提示 | "数据不存在或已被删除" | 返回列表页 |
| **数据冲突** | 并发编辑导致数据版本不一致 | 显示冲突提示，提供刷新选项 | "数据已被他人修改，请刷新后重试" | 用户选择刷新或覆盖 |
| **数据格式错误** | 返回数据格式不符合预期 | 显示错误提示，记录日志 | "数据格式异常，请联系管理员" | 提供反馈入口 |
| **数据加载失败** | 数据加载过程中出错 | 显示加载失败提示，提供重试 | "数据加载失败，请重试" | 用户点击重试 |

### 2.3 操作异常处理

| 异常类型 | 触发条件 | 处理方式 | 提示文案 | 后续操作 |
|---------|---------|---------|---------|---------|
| **操作失败** | 保存、删除等操作失败 | 显示失败提示，保留用户输入 | "操作失败：{具体原因}" | 用户修改后重试 |
| **重复操作** | 用户重复点击按钮 | 禁用按钮，防止重复提交 | - | 操作完成后恢复按钮 |
| **操作超时** | 操作执行时间过长 | 显示进度提示，提供取消选项 | "操作正在执行中..." | 用户可选择取消 |
| **前置条件不满足** | 操作的前置条件未满足 | 显示前置条件提示 | "请先完成{前置条件}" | 用户完成前置条件 |

## 四、数据状态管理规则库

### 4.1 状态定义模板

| 状态名称 | 状态说明 | 允许的操作 | 状态颜色 | 状态图标 |
|---------|---------|-----------|---------|---------|
| **草稿** | 数据未提交或未生效 | 编辑、删除、提交 | gray | edit |
| **生效** | 数据已生效，正在使用 | 查看、编辑(受限)、失效 | green | check-circle |
| **失效** | 数据已失效，不再使用 | 查看、删除 | red | times-circle |
| **审批中** | 数据正在审批流程中 | 查看、撤回 | orange | clock |
| **已拒绝** | 审批被拒绝 | 查看、重新提交 | red | ban |
| **已删除** | 数据已被删除(软删除) | 查看(历史)、恢复 | gray | trash |

### 4.2 状态流转规则

```
状态流转图模板：

[草稿] --提交--> [审批中] --审批通过--> [生效] --失效时间到--> [失效]
  |                |                        |
  └--删除--> [已删除]                      └--手动失效--> [失效]
                   |
                   └--审批拒绝--> [已拒绝] --重新提交--> [审批中]
```

### 4.3 状态约束规则

| 当前状态 | 允许的操作 | 禁止的操作 | 约束说明 |
|---------|-----------|-----------|---------|
| **草稿** | 编辑、删除、提交 | 生效、失效 | 草稿状态下可自由编辑 |
| **生效** | 查看、编辑(部分字段)、失效 | 删除 | 生效状态下只能编辑部分字段，不能删除 |
| **失效** | 查看、删除 | 编辑、生效 | 失效状态下只能查看和删除 |
| **审批中** | 查看、撤回 | 编辑、删除、生效 | 审批中不能修改数据 |
| **已拒绝** | 查看、重新提交 | 编辑、删除 | 已拒绝可重新提交 |

## 五、通用业务规则模板

### 5.1 删除规则模板

| 规则项 | 规则说明 | 检查时机 | 不满足时处理 |
|-------|---------|---------|-------------|
| **引用检查** | 检查是否被其他数据引用 | 删除前 | 提示"已被使用，无法删除" |
| **状态检查** | 检查数据状态是否允许删除 | 删除前 | 提示"当前状态无法删除" |
| **权限检查** | 检查用户是否有删除权限 | 删除前 | 提示"没有删除权限" |
| **确认机制** | 弹出确认对话框，防止误删 | 删除时 | 用户确认后才执行 |
| **软删除** | 标记为已删除，保留历史数据 | 删除时 | 数据标记为deleted=true |
| **硬删除** | 物理删除数据(谨慎使用) | 特定场景 | 彻底删除数据记录 |

### 5.2 编辑规则模板

| 规则项 | 规则说明 | 适用字段 |
|-------|---------|---------|
| **不可编辑字段** | 创建后不可修改的字段 | 编码、ID、创建人、创建时间等 |
| **受限编辑字段** | 特定状态下可修改的字段 | 生效状态下可修改备注、失效时间等 |
| **自由编辑字段** | 所有状态下都可修改的字段 | 备注、说明等辅助字段 |
| **编辑权限** | 只有创建人或管理员可编辑 | - |
| **版本管理** | 编辑时保留历史版本 | 重要数据 |

### 5.3 新增规则模板

| 规则项 | 规则说明 | 实现方式 |
|-------|---------|---------|
| **默认值设置** | 为字段设置合理的默认值 | 前端预设或后端自动填充 |
| **自动生成字段** | 系统自动生成的字段 | 编码、创建人、创建时间等 |
| **唯一性检查** | 检查关键字段的唯一性 | 提交前校验 |
| **必填校验** | 校验所有必填字段 | 提交前校验 |
| **关联数据加载** | 加载关联的基础数据 | 打开新增弹窗时 |

### 5.4 查询规则模板

| 规则项 | 规则说明 | 实现方式 |
|-------|---------|---------|
| **默认排序** | 默认按创建时间倒序排列 | ORDER BY createTime DESC |
| **筛选条件** | 提供常用筛选条件 | 费用组、费用类型、创建人等 |

## 七、使用指南

### 7.1 如何引用公共规则

在需求文档中引用公共规则时，使用以下格式：

```markdown
<!-- 示例1：引用校验规则 -->
<tr>
  <td>价格</td>
  <td>应用[数据校验规则-字段级]中的：必填校验、格式校验-数字、范围校验-非负、精度校验(2位小数)</td>
</tr>

<!-- 示例2：引用异常处理 -->
<tr>
  <td>保存失败</td>
  <td>应用[异常处理规则-操作异常]中的：操作失败处理规则</td>
</tr>

<!-- 示例3：引用权限控制 -->
<tr>
  <td>删除权限</td>
  <td>应用[权限控制规则-操作权限]中的：删除权限规则，角色：普通用户(禁止)</td>
</tr>
```

### 7.2 如何定制规则

当公共规则不满足特定需求时，可以在公共规则基础上定制：

```markdown
<!-- 定制示例 -->
<tr>
  <td>价格</td>
  <td>
    基础规则：应用[数据校验规则]中的必填校验、非负校验<br>
    定制规则：价格范围限制在0-10000之间(超出公共规则的范围校验)
  </td>
</tr>
```

### 7.3 规则优先级

当多个规则冲突时，按以下优先级处理：
1. **业务特定规则** > 公共规则
2. **字段级规则** > 表单级规则 > 业务级规则
3. **用户明确指定的规则** > 默认规则

## 八、维护指南

### 8.1 规则更新流程

1. 发现新的通用规则需求
2. 评估是否适合加入公共规则库
3. 在对应章节添加新规则
4. 更新版本号和更新日志
5. 通知团队成员规则变更

### 8.2 规则命名规范

- 使用简洁明确的名称
- 使用中文命名(便于理解)
- 遵循"{类型}-{具体规则}"的格式
- 示例："校验-必填"、"异常-网络超时"

|-------|---------|---------|-------|

**使用建议：**
1. 在编写PRD前，先查阅公共规则库
