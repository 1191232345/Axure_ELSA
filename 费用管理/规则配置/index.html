按钮交互逻辑

按钮 位置 触发动作 前置条件 后续操作
新增 筛选区域下方左侧 跳转到 rule-config-form.html 新建页面 无 打开新页面，表单为空，准备创建
查询 筛选区域右侧 执行筛选，重新渲染表格 无 根据4个筛选条件过滤ruleConfigs并刷新
重置 筛选区域查询按钮左侧 清空所有筛选条件，恢复全量显示 无 4个筛选项清空，renderRuleConfigTable()无参数调用
查看 操作列 打开详情模态框 规则配置存在 显示完整信息：基础信息、折扣配置明细、生效周期、调整记录
编辑 操作列 跳转到 rule-config-form.html?editId=xxx 规则配置存在 打开新页面，自动填充已有数据到表单和折扣配置
删除 操作列 弹出确认对话框，确认后删除 规则配置存在 confirm确认→deleteRuleConfig()→saveRuleConfigs()→刷新表格→Toast成功提示
字段 中文名 类型 长度 必填 说明
id规则配置IDString(UUID)36是系统生成，唯一标识
name配置名称String128是默认=客户名-仓库名-规则配置
customer_id客户IDString36是关联客户表，+warehouse_id唯一
customer_name客户名称String64冗余根据customer_id关联查询
warehouse_id仓库IDString36是关联仓库表，+customer_id唯一
warehouse_name仓库名称String64冗余根据warehouse_id关联查询
price_card_id价卡IDString36是关联价卡表
price_card_name价卡名称String128冗余根据price_card_id关联查询
business_type业务类型Enum16是all/inbound/outbound/transfer/return
fee_discounts折扣配置JSON Object-是按分类存储：{inbound:[], outbound:[], storage:[], express:[], other:[]}
adjustments调整记录JSON Array-否从价卡继承的调整项 [{type, name, amount, description}]
is_adjusted微调状态Boolean-计算对比价卡原始折扣与当前折扣是否一致
effective_start_time生效开始时间DateTime-是格式 YYYY-MM-DD HH:mm
effective_end_time生效结束时间DateTime-是需大于生效开始时间
created_by创建人String64是手工输入
created_at创建时间DateTime-系统创建时自动生成 getCurrentDateTime()
updated_at更新时间DateTime-系统每次保存时自动更新
校验项 校验规则 触发时机 错误提示
必填校验 name/customer_id/warehouse_id/price_card_id/effective_start_time/effective_end_time/created_by 均不能为空 保存时（createRuleConfig / updateRuleConfig） "请填写所有必填项"
唯一性校验 同一 customer_id + warehouse_id 组合只能有一个有效配置（未过期） 选择客户/仓库时实时检测 + 保存时再次校验 "该客户和仓库已存在配置：xxx"
时间范围校验 effective_end_time 必须 > effective_start_time 保存时 前端未强制校验，建议补充
微调状态计算 逐字段对比 fee_discounts 与 price_card.fee_discounts 保存时 checkIfAdjusted() 任一字段差异则 is_adjusted=true
