字段 说明 数据来源 取值规则 显示格式 存储长度 属性存储类型
客户代码 规则配置关联的客户 手工选择（客户列表） 必填，从活跃客户中选择 下拉选择框 64 字符型

价卡模板 引用的基础价卡，作为折扣配置模板 手工选择（价卡列表） 必填，选择后自动加载折扣配置 下拉选择框 64 字符型

失效时间 规则配置失效的时间 手工输入 可选，不能早于生效时间 日期时间选择器 32 时间戳
费用类型 折扣配置中的费用类型项 价卡模板数据 + 手工选择 按费用分类Tab分组显示，可选 下拉选择（按子类分组） 64 字符型
折扣方式 该费用类型的折扣计算方式 弹窗配置 / 系统带出 默认=未配置，通过配置弹窗设置 胶囊徽章展示 32 字符型
备注 费用项的附加说明 手工输入 可选，用于补充说明 文本输入框 256 字符型

交互逻辑

触发条件 交互逻辑 展示内容
选择客户/仓库 1. 自动生成默认名称：客户名-仓库名-规则配置
2. 实时检测是否已存在相同客户+仓库的组合
3. 如重复则显示警告提示		 名称输入框自动更新，重复提示区域显示/隐藏
选择价卡模板 1. 加载价卡的折扣配置数据
2. 将折扣配置按分类转换为feeRows
3. 显示折扣配置区域
4. 渲染费用分类Tab和表格		 折扣配置区域展开，显示费用分类Tab和已加载的费用行
切换费用分类Tab 1. 切换Tab激活状态
2. 过滤显示当前分类下的费用行
3. 更新费用类型下拉框选项		 Tab样式切换，表格内容过滤更新
点击"新增行"按钮 1. 在当前费用分类下添加一行空记录
2. 新行默认discount_type=none
3. 重新渲染表格		 费用表格底部新增一行空白行
点击"配置"按钮 1. 将当前行数据注册到AppState（桥接价卡modal）
2. 标记规则配置模式(_isRuleConfigMode)
3. 根据是否有阶梯定价决定打开标准/阶梯折扣弹窗		 弹出折扣配置弹窗（与价卡管理一致），支持阶梯价格预览
保存折扣配置 1. 验证折扣值有效性
2. 回写折扣数据到规则配置的feeRows
3. 关闭弹窗
4. 重新渲染表格，更新胶囊徽章		 弹窗关闭，折扣方式列显示更新后的胶囊徽章
点击"保存配置"按钮 1. 校验所有必填字段
2. 校验客户+仓库唯一性
3. 收集表单数据和折扣配置
4. 计算是否与原价卡有调整(is_adjusted)
5. 创建/更新规则配置记录
6. 跳转回列表页		 Toast提示成功，0.8秒后跳转列表页
点击"取消"按钮 跳转回列表页，不保存任何数据 直接返回列表页
逻辑项 说明 数据来源 展示规则 备注
弹窗打开 点击"配置"按钮打开折扣配置弹窗 前端逻辑 弹窗居中显示，背景遮罩 复用价卡管理的StandardDiscountModal/TierDiscountModal
数据桥接 将规则配置行的数据临时注册到AppState.feeRows 当前feeRows + AppState 通过row.id匹配，设置_isRuleConfigMode标记 保存时通过标记回写到规则配置feeRows
弹窗类型判断 根据费用项是否有original_tier_pricing数据判断 当前行的original_tier_pricing字段 有阶梯数据→TierDiscountModal，无→StandardDiscountModal 与价卡管理FeeTable.showDiscountConfig逻辑完全一致
价格预览 始终使用阶梯式价格预览展示 DiscountCalculator.calculateFinalPrice 显示阶梯段、原价格、折扣、折后价格 无阶梯数据时将单一价格作为默认阶梯
字段 说明 数据来源 取值规则 显示格式 备注
折扣方式 选择折扣的计算模式 手工选择 单选：无折扣、折扣率、增减、一口价 下拉选择框 必选，默认"无折扣"
折扣值 输入具体的折扣数值 手工输入 数字输入框，支持小数点后两位 输入框 + 单位（% 或 $） 选择"无折扣"时隐藏
阶梯段 显示费用的阶梯范围 原始阶梯定价数据 只读展示 如 "1-100件"、"101-∞" 只读
原价格 每个阶梯的原始单价 原始阶梯定价数据 只读展示 如 "$10.00" 只读
折后价格 应用折扣后的最终价格 DiscountCalculator实时计算 根据折扣方式和折扣值动态计算 金色高亮 "$9.00" 不允许为负数
触发条件 交互逻辑 展示内容
切换折扣方式 1. 显示/隐藏折扣值输入框
2. 切换单位标签（%/$）
3. 实时重新计算预览价格		 折扣值区域显隐切换，单位变化，价格预览刷新
修改折扣值 每次输入变化触发实时计算，更新所有阶梯的折后价格 价格预览表格中的折后价格列实时更新
点击"保存"按钮 1. DiscountCalculator.validateConfig验证
2. 写入tierDiscountConfig到AppState行
3. 因_isRuleConfigMode标记，同步回写规则配置feeRows
4. 关闭弹窗并重新渲染表格		 弹窗关闭，表格中对应行的胶囊徽章更新
点击"取消"按钮 关闭弹窗，不保存任何更改 弹窗关闭，表格数据不变
点击遮罩层 关闭弹窗（等同于取消） 弹窗关闭
